Preface

What is this book about?

This book is a sci-fi documentary about how technologies - which are within our grasp today - have transformed software development in the 2040s. That's right, we're going to look 20 years into the future to get a taste for what's coming!

Why? Well, we're Stuck!

We were promised IDEs that write code for us; all we got was "dark mode"!

My reading of old texts gives me the impression that when it comes to the tools, technologies, and processes we use daily to create software there hasn't been much progress in the last 20-50 years.
This is a counter-intuitive claim so let's take a look at some examples:
- IDEs and code editors are painfully outdated: Emacs is 50 years old and IDEs with features comparable to what we have today appeared in the early 1970s.
- Revision control tools haven't seen major innovations since git was introduced, and git captures only a fraction of the context around changes in software.
- Input devices (mouse, keyboard) haven't changed for 50 years.
- User interfaces (GUIs or command line) for our desktop environments are also 50 years old.
- Software development methodologies like Scrum and XP are 25 years old (refining ideas from the 1950s), and we still disagree on what "agile" means leading to some of its founders' declaration that "agile is dead"!
- Best practices: TDD (mid 90s), Pragmatic Programmer (1999), Clean Code (2009) and other collections of best practices have great potential but we have no way of enforcing their use.
- We don't know what makes developers productive, let alone have tools for measuring it.
Now let's consider what happened during the first couple of decades after computers became useful in the 1940s. In the beginning, Programmers tested programs by doing calculations by hand. To implement programs they punched holes onto paper. Within 2 decades they had invented human readable programming languages, compilers, editors, debuggers, IDEs, GUIs, and operating systems. These inventions made those programmers several thousand times more productive. These days the most ambitious developers chase 10X productivity gains!

Let's get to work!

You might be wondering what's behind this slow-down. There are different reasons, but I think the main reason is lack of vision. Therefore, this book is a vision of a better future, an exciting future not about incremental improvements, but revolutionary ones. A future you'd be excited to live in. A future so great you'd want to get busy now to make it possible.
